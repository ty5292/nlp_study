# 作业1：Joint BERT 代码阅读报告

## 一、BERT 文本分类和实体识别有什么关系？分别使用什么 loss？

在 Joint BERT 模型中，文本分类（Intent Classification）和实体识别（Slot
Filling / Named Entity Recognition）是两个同时进行的任务，它们共享同一个
BERT 编码器，通过多任务学习（Multi-task Learning）的方式共同训练。

具体来说，输入的文本首先经过 BERT
编码器得到上下文表示，然后分别用于两个不同的任务。

### 1. 文本分类（Intent Classification）

文本分类任务用于识别整个句子的意图。例如在语音助手或对话系统中，判断用户是想查询天气、播放音乐还是查询路线。

在 BERT 模型中，通常使用句子开头的 **\[CLS\] 向量**
作为整句话的语义表示，并将其输入到全连接层进行分类预测。

常用的损失函数：

**CrossEntropy Loss（交叉熵损失）**

公式：

L_seq = - Σ y_i log(p_i)

其中：

-   y_i 表示真实标签
-   p_i 表示模型预测概率

### 2. 实体识别（Slot Filling / NER）

实体识别任务是对句子中的每一个 token
进行标注，用于识别文本中的关键信息，例如地点、时间、人物等。

BERT 输出每个 token 的向量表示，然后通过分类层对每个 token
进行标签预测。

常用的损失函数包括：

-   CrossEntropy Loss
-   CRF Loss（条件随机场损失）

如果不使用 CRF，则直接对每个 token 的分类结果计算 CrossEntropy Loss；\
如果使用 CRF，则通过 CRF 层建模标签之间的依赖关系，并计算 CRF Loss。

因此，Joint BERT 通过共享 BERT 编码器，同时完成：

-   **句子级任务（Intent Classification）**
-   **序列标注任务（Slot Filling）**

从而利用两个任务之间的语义信息，提高整体模型效果。

## 二、多任务训练 loss = seq_loss + token_loss 有什么坏处？如果存在训练不平衡的情况，如何处理？

在 Joint BERT 的训练过程中，通常会将两个任务的损失函数直接相加：

    Loss = seq_loss + token_loss

其中：

-   **seq_loss**：文本分类任务的损失\
-   **token_loss**：实体识别任务的损失

这种简单相加的方法虽然实现简单，但也存在一些问题。

### 1. 任务训练不平衡

由于两个任务的损失值范围可能不同，例如：

    seq_loss ≈ 0.2
    token_loss ≈ 5.0

此时总 loss 会主要受到 token_loss
的影响，模型在训练过程中会更加关注实体识别任务，而忽略文本分类任务，从而导致训练不平衡。

### 2. 标签数量不同

在文本分类任务中，一句话只有 **一个标签**；\
而在实体识别任务中，一句话可能包含 **多个 token 标签**。

因此 token_loss 的计算次数远多于
seq_loss，也会导致模型更加偏向优化实体识别任务。

### 3. 梯度冲突问题

在多任务学习中，不同任务的优化目标可能存在冲突。如果两个任务的梯度方向不一致，简单相加可能会导致训练过程不稳定，甚至影响模型的收敛速度和最终效果。

------------------------------------------------------------------------

## 三、解决训练不平衡的方法

### 1. Loss 加权

最常见的方法是为不同任务的损失设置权重：

    Loss = α × seq_loss + β × token_loss

其中：

-   α 表示文本分类任务权重\
-   β 表示实体识别任务权重

通过调节权重，可以使两个任务对模型训练的影响更加平衡。

### 2. 动态权重调整

可以根据训练过程中任务的难度或不确定性，动态调整各任务的权重。例如使用
**Uncertainty Weighting** 方法，让模型自动学习最合适的损失权重。

### 3. 梯度平衡方法

例如 **GradNorm**
方法，通过调整任务梯度大小，使不同任务的训练速度保持一致，从而实现训练平衡。

### 4. 交替训练

在训练过程中可以交替优化不同任务，例如：

-   在部分 batch 中训练文本分类任务\
-   在部分 batch 中训练实体识别任务

以减少任务之间的干扰。
